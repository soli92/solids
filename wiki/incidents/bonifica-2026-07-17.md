## Incidente 3 â€” Struttura git corrotta

**Classificazione**: rischio di perdita dati / confusione sviluppo

### Submodule soli-projects[^src: docs/raw/bonifica-repo-2026-07-17.txt Â§Sezione 4 â€” Struttura git corrotta]

Il file `.gitmodules` referenziava `soli-projects` come submodule, ma il repository
puntato risultava non raggiungibile / obsoleto.

**Azione**:
```bash
git submodule deinit -f soli-projects
git rm -f soli-projects
rm -rf .git/modules/soli-projects
```

### Clone innestato solids/

Nella root del repository esisteva `solids/` â€” un clone git separato del medesimo
repository (clone accidentale, non submodule registrato). Contenuto confermato come
pulito (stesso contenuto del remote), quindi eliminato con `rm -rf solids/`.

`.gitignore` aggiornato con `/solids/` (root-anchored) per prevenire il problema in futuro.

**Decisione**: [[da-003-gitignore-root-anchored]] â€” pattern `/solids/` con slash iniziale
evita di ignorare anche `registry/solids/`.
