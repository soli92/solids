# Getting Started

Come usare **SoliDS** in un progetto in pochi passaggi.

## 1. Installazione

```bash
npm install @soli92/solids
```

## 2. Importa il CSS

Entrypoint completo (consigliato):

```css
@import "@soli92/solids/css/index.css";
```

Include variabili, override per **dark**, **fantasy**, **cyberpunk**, **90s-party**, **steampunk** e temi **personaggio**, layer shadcn, base e utilities. I default **light** / **dark** seguono una lettura **Material Design 3** (superfici tonali, raggi generosi, ombre a livelli), con primary blu allineato a shadcn. Ordine e contenuti: pagina **Foundations / Tokens**.

## 3. Scegli il tema

Imposta lŌĆÖattributo su `<html>` (o `:root`):

```html
<html data-theme="light"></html>
<html data-theme="dark"></html>
<html data-theme="fantasy"></html>
<html data-theme="cyberpunk"></html>
<html data-theme="90s-party"></html>
<html data-theme="steampunk"></html>
```

Senza `data-theme`, il **dark** di sistema (`prefers-color-scheme`) applica i token dark; **fantasy**, **cyberpunk**, **90s-party**, **steampunk** e i temi **personaggio** restano espliciti. Dettagli: **Foundations / Themes**.

Con **next-themes**, usa `attribute="data-theme"` e valori come in `docs/shadcn-integration.md` (inclusi i temi **personaggio**). Per comparire tutti nel selettore, estendi `themes` su `ThemeProvider`.

## 4. Tailwind + shadcn (opzionale ma consigliato)

```js
// tailwind.config.cjs (o .js)
module.exports = {
  presets: [require("@soli92/solids/tailwind-preset")],
  content: ["./app/**/*.{ts,tsx}", "./components/**/*.{ts,tsx}"],
};
```

Poi `npx shadcn@latest add ŌĆ”` con gli item del registry `@solids`. Guida: `docs/shadcn-integration.md`.
