# Roadmap

## Stato attuale

✅ Design tokens (base + semantic + temi **light**, **dark**, **fantasy**, **cyberpunk**, **90s-party**, **steampunk**, sei temi **personaggio**)  
✅ Verifiche automatiche post-build: **`npm run test:tokens`** (`scripts/tokens-sanity.mjs` su `dist/tokens/tokens.json` + `variables.css`)  
✅ CSS variables e pipeline `index.css` (inclusi default MD3-oriented per light/dark)  
✅ Fallback OS dark senza `data-theme`  
✅ Base styles e utility CSS (incl. `.sd-font-heading`, motion `standard` / `emphasized-*`)  
✅ Preset Tailwind (`fontFamily.heading`, `dark:` per temi scuri nominati) e layer shadcn  
✅ Storybook: foundations + **SoliDS/UI**; toolbar **Tema DS** (tutti i temi); `npm test` = build + static SB; [GitHub Pages](https://soli92.github.io/solids/) con le release  
✅ Registry shadcn (modello 1, kit `solids-ui`)  

## Prossimi step

- Esempi e linee guida accessibilità più estese
- Adapter / snippet per altri framework oltre a React (documentazione)
- Evoluzione continua delle story e del registry

## Visione

SoliDS vuole restare una base **solida, estendibile e manutenibile** per prodotti reali, con token e temi coerenti indipendentemente dallo stack UI.
