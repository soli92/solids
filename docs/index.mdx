# SoliDS

**SoliDS** è un **Design System agnostico** basato su **design tokens** e **CSS**.

Nasce per fornire fondamenta solide, coerenti e riutilizzabili per la costruzione di interfacce utente, indipendentemente dal framework o dallo stack tecnologico utilizzato.

## Obiettivi

- Centralizzare le decisioni di design
- Garantire coerenza visiva e comportamentale
- Ridurre ambiguità e ridondanze
- Abilitare un’adozione progressiva

SoliDS non è una component library, ma la base su cui **componenti e applicazioni possono essere costruiti**.

## Cosa include

- Design tokens (JSON)
- CSS variables semantic‑first
- Supporto nativo a light / dark theme
- Base styles e utility CSS

## Cosa non include (per ora)

- Componenti UI
- Dipendenze da framework specifici
