# SoliDS

**SoliDS** è un **Design System agnostico** basato su **design tokens** e **CSS**.

Nasce per fornire fondamenta solide, coerenti e riutilizzabili per la costruzione di interfacce utente, indipendentemente dal framework o dallo stack tecnologico utilizzato.

## Obiettivi

- Centralizzare le decisioni di design
- Garantire coerenza visiva e comportamentale
- Ridurre ambiguità e ridondanze
- Abilitare un’adozione progressiva

SoliDS non pubblica una component library “chiusa” nell’artifact npm: il pacchetto `@soli92/solids` offre **token + CSS**; i componenti UI vivono nei tuoi progetti (modello **shadcn**: codice in repo + registry).

## Cosa include

- Design tokens (JSON unificato: base, semantic, **sei temi**)
- CSS variables semantic‑first (`--sd-*`)
- **Temi:** light, dark, **fantasy**, **cyberpunk**, **90s-party**, **steampunk**, sei ispirati a **personaggi** anime/comics (`data-theme` su `<html>`)
- Fallback **OS dark** quando `data-theme` non è impostato
- Base styles e utility CSS (`sd-*`)
- Preset Tailwind per allineamento con **shadcn/ui**
- **Modello 1 (shadcn):** registry `@solids`, JSON in `registry/r/`; guida in [docs/registry-model-1.md](https://github.com/soli92/solids/blob/main/docs/registry-model-1.md)
- **Storybook:** questa documentazione (sidebar ordinata: home → guide → Foundations → UI) e le story **SoliDS/UI**; toolbar **Tema DS** per anteprima token su tutti i temi; **online** dopo ogni release su [soli92.github.io/solids](https://soli92.github.io/solids/). In repo: `npm run storybook`; verifica statica: `npm test`

## Cosa non include nell’artifact npm

- I file React in `src/components/ui/` non sono nel tarball npm: sono riferimento e **registry**; nei progetti si aggiungono con la CLI shadcn dagli item `@solids/…`.
