Il repository `solids` presentava quattro categorie di problemi critici:

1. **Icone duplicate** — 9 icone tematiche definite due volte (versioni semplificate in `glyphs.tsx` + versioni elaborate in `glyphs-themed.tsx`, non esportate)
2. **Import path rotto** — `logo-loader.tsx` importava da `@/icons` invece di `@/lib/icons`, path non risolvibile dai consumer del registry shadcn
3. **Struttura git corrotta** — submodule `soli-projects` defunto + clone innestato `solids/` nella root
4. **Adapter Claude Code assente** — `.claude/` parziale (solo `share.md` + `content-share-protocol.md`), nessun agente, nessun CLAUDE.md

Tutti i problemi sono stati risolti atomicamente in un unico commit di bonifica.

## Sezione 1 — Design System: icone duplicate

### Problema

Due file convivevano con definizioni degli stessi simboli:

- `src/icons/glyphs.tsx` — conteneva versioni **semplificate** (segnaposto) di 9 icone tematiche aggiunge per errore
- `src/icons/glyphs-themed.tsx` — conteneva le versioni **elaborate** (design definitivo) degli stessi simboli, ma non era esportato

Le 9 icone duplicate erano:
`IconFantasyScroll`, `IconFantasySword`, `IconFantasyGem`, `IconCyberpunkChip`, `IconCyberpunkEye`, `IconCyberpunkSignal`, `IconPartyBolt`, `IconPartyDiamond`, `IconPartyStar`

Se entrambi i file fossero stati esportati, TypeScript avrebbe generato errori di binding duplicato.

### Root cause

`src/icons/index.ts` esportava solo `glyphs.tsx` e `solids-icon.tsx`. Il file `glyphs-themed.tsx` era stato creato con le implementazioni corrette ma mai aggiunto all'index, lasciando le icone inaccessibili.

### Fix applicato

1. Rimosso righe 247–342 da `src/icons/glyphs.tsx` (le 9 versioni semplificate)
2. Aggiunto `export * from "./glyphs-themed"` in `src/icons/index.ts`

Decisione: conservare le versioni di `glyphs-themed.tsx` perché hanno SVG path elaborati corrispondenti al design definitivo; le versioni in `glyphs.tsx` erano segnaposto.

## Sezione 2 — Design System: import path rotto

### Problema

`src/components/ui/logo-loader.tsx` importava:

```
import { IconSoli4x3WithTextTheme } from "@/icons"
```

Questo funziona in locale (dove `@/` → `src/`), ma i consumer del registry shadcn ricevono i file via `registry:lib` installati in `lib/`. Il path `@/icons` non esiste nel progetto del consumer; il corretto sarebbe `@/lib/icons`.

### Fix applicato

1. Creato `src/lib/icons/index.ts` — bridge che re-esporta da `@/icons` per compatibilità dev locale
2. Modificato `logo-loader.tsx`: `@/icons` → `@/lib/icons`

Il bridge garantisce che in locale dev il path `@/lib/icons` risolva correttamente senza rompere la toolchain esistente.

### Problema

- `registry/solids/icons/glyphs-themed.tsx` non esisteva (non era mai stato sincronizzato)
- `registry/r/solids-icons.json` non esisteva (le icone tematiche non avevano un registry item dedicato)

### Fix applicato

Nota: `sync-registry.mjs` rigenera `registry.json` ad ogni run — qualsiasi modifica manuale viene sovrascritta. Le modifiche devono sempre andare nello script.

## Sezione 4 — Struttura git corrotta

### Submodule soli-projects

Il file `.gitmodules` referenziava `soli-projects` come submodule, ma il repository puntato risultava non raggiungibile / obsoleto. La directory non era più sincronizzata con il remote.

**Azione**: rimozione completa del submodule.

```bash
git submodule deinit -f soli-projects
git rm -f soli-projects
rm -rf .git/modules/soli-projects
```

### Clone innestato solids/

Nella root del repository esisteva una directory `solids/` che era un clone git separato del medesimo repository (clone accidentale). Non era un submodule registrato; era semplicemente una copia della working tree.

Confermato che la directory era pulita (stesso contenuto del remote, nessun lavoro locale esclusivo), quindi eliminata con `rm -rf solids/`.

Il `.gitignore` è stato aggiornato con il pattern root-anchored `/solids/` per impedire che future clonazioni accidentali finiscano tracciate. Il pattern non usa il prefisso `solids/` (senza slash iniziale) che avrebbe ignorato anche `registry/solids/`.

## Decisioni architetturali

### DA-001 — Conservare glyphs-themed.tsx come source of truth

Le versioni elaborate in `glyphs-themed.tsx` sono il design definitivo. Le versioni semplificate in `glyphs.tsx` erano placeholders. Decisione: eliminare le versioni semplificate, non le elaborate.

### DA-002 — Bridge pattern per @/lib/icons

Invece di cambiare il path sorgente (che richiederebbe refactoring in tutto il codebase), si introduce un bridge `src/lib/icons/index.ts` che re-esporta da `@/icons`. Questo separa il path del consumer del registry (stabile) dalla struttura interna del repo (flessibile).

### DA-003 — Pattern .gitignore root-anchored

Il pattern `/solids/` con slash iniziale ignora solo la directory nella root del repo. Il pattern `solids/` senza slash avrebbe ignorato anche `registry/solids/`, causando la scomparsa delle icone sincronizzate dal tracking git. Sempre usare slash iniziale per pattern nella root.

### DA-004 — .claude/worktrees/ gitignored

La directory `.claude/worktrees/` usata da Claude Code per checkout isolati deve essere ignorata da git. Aggiunto pattern nel `.gitignore`. Questi checkout sono temporanei e non devono essere committati.

## Impatto e rischi residui

**Risk residuo**: il TypeScript build (`npm run build`) non è stato eseguito in questa sessione per verificare zero errori. Raccomandato come passo di verifica prima della prossima release.
