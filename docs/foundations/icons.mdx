# Icons

SoliDS include un sistema di icone SVG React basato su **`SolidsIcon`** — un wrapper che applica i token `--sd-color-icon-*` automaticamente tramite CSS variables, seguendo il tema attivo su `data-theme`.

> **v1.13.1** — aggiunto anche il **Soli brand icon pack** (8 varianti) con versioni mono, gold e theme-aware per tutti i temi SoliDS. Tutte le icone esportate da `@soli92/solids/icons`.

## Utilizzo base

```tsx
import { IconHome, IconStar, IconAlertCircle } from "@soli92/solids/icons";

<IconHome size="md" variant="default" />
<IconStar size="lg" variant="primary" />
<IconAlertCircle size="sm" variant="muted" />
```

## Prop `variant`

| Variant | Token CSS | Uso consigliato |
|---------|-----------|-----------------|
| `default` | `--sd-color-icon-default` | Icone standard su sfondo canvas/surface |
| `muted` | `--sd-color-icon-muted` | Icone secondarie, placeholder, decorative |
| `primary` | `--sd-color-icon-primary` | Icone in evidenza, azioni principali |
| `on-primary` | `--sd-color-icon-on-primary` | Icone su sfondo primary (es. button filled) |

Tutti i temi definiscono `color.icon.*` — il colore si adatta automaticamente al `data-theme` attivo.

## Prop `size`

| Size | px |
|------|----|
| `sm` | 16px |
| `md` | 20px |
| `lg` | 24px |
| `xl` | 32px |
| `number` | valore custom (es. `size={48}`) |

## Icone tema-specifiche

Tre set di icone SVG disegnati per rispecchiare l'identità visiva dei temi fantasy, cyberpunk e 90s-party.

|-------|-------------|

| `IconFantasySword` | Spada medievale diagonale con elsa e pomello circolare |

## Usare icone nel tema corretto

Le icone tema-specifiche funzionano con qualsiasi tema (usano `--sd-color-icon-*`), ma danno il meglio abbinate al tema corrispondente:

```tsx
<section data-theme="cyberpunk" style={{ background: "var(--sd-color-bg-canvas)" }}>
  <IconCyberpunkEye variant="primary" size="xl" />
</section>
```

## Aggiungere icone personalizzate

```tsx
import * as React from "react";
import { SolidsIcon, type SolidsIconProps } from "@soli92/solids/icons";

type IconProps = Omit<SolidsIconProps, "children">;

const IconMyCustom = React.forwardRef<SVGSVGElement, IconProps>(
  function IconMyCustom(props, ref) {
    return (
      <SolidsIcon ref={ref} {...props}>
        {/* viewBox="0 0 24 24", stroke="currentColor" già impostati */}
        <path d="M12 2L2 7l10 5 10-5-10-5z" />
        <path d="M2 17l10 5 10-5" />
        <path d="M2 12l10 5 10-5" />
      </SolidsIcon>
    );
  }
);
```

## Token `color.icon.*`

Tutti i temi definiscono questi quattro token:

```json
"icon": {
  "default":    "...",
  "muted":      "...",
  "primary":    "...",
  "on-primary": "..."
}
```

CSS variables generate:
- `--sd-color-icon-default`
- `--sd-color-icon-muted`
- `--sd-color-icon-primary`
- `--sd-color-icon-on-primary`

Anteprima interattiva con theme selector → **Foundations / Icons → Interactive**.
