# Themes

SoliDS applica i temi con **override dei token semantici** (colori, font, raggio, ombre): gli stessi nomi di variabile CSS (`--sd-*`) cambiano valore in base a `data-theme` su `<html>` (o `:root`).

## Temi disponibili

| Tema | `data-theme` | Note |
|------|----------------|------|
| **Light** | `light` oppure default `:root` | Base chiara; corpo **Inter**, titoli **DM Sans**, mono **JetBrains Mono**. |
| **Dark** | `dark` | Contrasto elevato; stessi stack font del light su superfici scure. |
| **Fantasy** | `fantasy` | Pergamena, inchiostro, ottone; corpo **Source Serif 4**, titoli **Cinzel**; mono **JetBrains Mono**. |
| **Cyberpunk** | `cyberpunk` | Sfondo profondo, accenti cyan/magenta; corpo **Space Grotesk**, titoli **Orbitron**, mono **Share Tech Mono**. |
| **90s party** | `90s-party` | Viola/magenta/teal/lime rave; titoli **Russo One**, corpo **Tahoma/Verdana**, mono **VT323**. |
| **Steampunk** | `steampunk` | Cuoio/carbone, ottone e rame; titoli **Cinzel**, corpo **Libre Baskerville**, mono **Courier Prime**. |
| **Ichigo** | `ichigo` | Nero/arancio-rosso (*Bleach* / Bankai); titoli **Bebas Neue**. |
| **Vegeta** | `vegeta` | Blu royal e oro (*Dragon Ball*); titoli **Rajdhani**. |
| **Zoro** | `zoro` | Verde profondo e argento (*One Piece*); titoli **Merriweather**. |
| **Captain America** | `captain-america` | Blu notte, rosso e bianco (MCU); titoli **Oswald**. |
| **Sasuke** | `sasuke` | Notte viola/indaco (*Naruto* / Chidori); titoli **Rajdhani**. |
| **Inuyasha** | `inuyasha` | Rosso haori, argento lunare, terra (*Inuyasha*); titoli **Crimson Text**, corpo **Merriweather**. |

### Comportamento con il sistema

- Se **`data-theme` non è impostato**, `prefers-color-scheme: dark` applica i token **dark**.
- Con temi espliciti (fantasy, cyberpunk, ecc.) il tema resta quello scelto indipendentemente dalla preferenza OS.

### Tailwind `dark:`

Il preset Tailwind tratta come "dark": `dark`, `cyberpunk`, `90s-party`, `steampunk`, `captain-america`, `ichigo`, `inuyasha`, `sasuke`, `vegeta`, `zoro`.

## Cosa resta stabile

- Nessuna duplicazione di componenti: cambiano solo i **token**.
- Il layer **shadcn** continua a leggere `--background`, `--primary`, ecc., già mappati sui token SoliDS.

## Attivazione

```html
<html data-theme="light"></html>
<html data-theme="dark"></html>
<html data-theme="fantasy"></html>
<html data-theme="cyberpunk"></html>
<html data-theme="90s-party"></html>
<html data-theme="steampunk"></html>
<html data-theme="ichigo"></html>
<html data-theme="vegeta"></html>
<html data-theme="zoro"></html>
<html data-theme="captain-america"></html>
<html data-theme="sasuke"></html>
<html data-theme="inuyasha"></html>
```

Anteprima interattiva nella story **Foundations / Themes → Token Detail**.
