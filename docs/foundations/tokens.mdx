# Design Tokens

I token sono la **singola fonte di verità** per colori, spazi, tipografia, raggio, ombre, motion e z-index. SoliDS li espone come:

1. **JSON** — per tooling, documentazione e integrazioni (`dist/tokens/tokens.json`)
2. **CSS variables** — prefisso `--sd-*`, consumabili da qualsiasi stack

## Struttura del JSON pubblicato

`tokens.json` unisce:

- **`base`** — palette grezza, scala spacing numerica, radius base, font base, shadow, easing, z-index, …
- **`semantic`** — significati (testo, sfondo, border, intent, primary, …) con default “light”
- **`themes`** — override per ogni file in `src/tokens/themes/*.json` (**`light`**, **`dark`**, **`fantasy`**, **`cyberpunk`**, **`90s-party`**, **`steampunk`**, temi **personaggio**, …)

Esempio di accesso in JavaScript:

```js
import tokens from "@soli92/solids/tokens";

// tokens.semantic, tokens.base, tokens.themes.dark, …
```

## Pipeline CSS (ordine di import)

L’entrypoint consigliato è `@soli92/solids/css/index.css`, che importa nell’ordine:

1. **`variables.css`** — `:root` con base + semantic + merge tema light
2. **`themes.css`** — un blocco `[data-theme="…"]` per ogni tema (dark, fantasy, cyberpunk, 90s-party, steampunk, personaggio, …), più fallback `prefers-color-scheme: dark` quando `data-theme` non è impostato
3. **`shadcn.css`** — mapping verso variabili shadcn (`--background`, `--primary`, …)
4. **`base.css`** — reset e stili globali
5. **`utilities.css`** — classi `sd-*`

## Motion (base)

In `base.json`: durate (`--sd-duration-*`) e curve **MD3-friendly** (`--sd-easing-standard`, `--sd-easing-emphasized-decelerate`, `--sd-easing-emphasized-accelerate`) oltre a `ease-in` / `ease-out` / `ease-inout`.
