# Border Radius

I border radius sono esposti come **CSS variables** `--sd-radius-*`, da `src/tokens/base.json` con override in **`semantic.json`** e nei JSON tema. Per **light** / **dark** i passi semantici tipici sono **8px / 12px / 16px / 20px** (sm → xl), in linea con componenti **Material Design 3**; **fantasy**, **cyberpunk**, **90s-party** e **steampunk** possono **sovrascrivere** (es. fantasy più morbido, cyberpunk/steampunk più “tech” o meccanici).

## Token CSS (default light / base)

| Variabile | Ruolo |
| --------- | ----- |
| `--sd-radius-none` | Nessun raggio (0) |
| `--sd-radius-sm` | Piccolo |
| `--sd-radius-md` | Standard (shadcn usa spesso questo come `--radius`) |
| `--sd-radius-lg` | Grande |
| `--sd-radius-xl` | Extra large |
| `--sd-radius-2xl` | Molto arrotondato |
| `--sd-radius-3xl` | Ancora più ampio |
| `--sd-radius-full` | Pill / cerchio |

## Utility

Le classi in `utilities.css` includono ad esempio `.sd-rounded-sm`, `.sd-rounded-md`, `.sd-rounded-lg`, `.sd-rounded-xl`, `.sd-rounded-full`, `.sd-rounded-none`. In **Tailwind** con il preset SoliDS, `rounded-lg` / `rounded-md` / `rounded-sm` dei componenti shadcn seguono `--radius` → `--sd-radius-md` del tema attivo.

## Linee guida

- Parti dal token più piccolo adeguato al contesto
- Evita valori arbitrari nei componenti; allinea i componenti simili allo stesso step
- Per shadcn/ui, `--radius` nel layer `shadcn.css` punta a `--sd-radius-md` e segue i token del tema attivo
